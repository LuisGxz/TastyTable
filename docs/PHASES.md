# TastyTable — Estado de fases

> App móvil de descubrimiento y reserva de restaurantes + panel del restaurante. "Apetitosa y elegante".
> Leer al iniciar sesión de trabajo. Vara de calidad: **FinPulse** (NO NEGOCIABLE — PORTFOLIO_PROJECTS.md §2
> "Quality gate"), incl. **Capa de Demo Guiada role-aware**.
> **Deploy diferido**: se construye completo menos el deploy en la nube (en bloque al final, con el resto).

## Stack
- **Front:** **Ionic 8 + Angular 20** (standalone + signals), **PWA instalable**. Mobile-first, gestos, transiciones suaves.
- **Back:** **NestJS 11** REST (Mongoose). Lógica de disponibilidad por fecha/hora/personas/zona.
- **DB:** **MongoDB 7** (Mongoose 9). Vía Docker (`docker-compose.yml`, puerto **27018**).
- **Auth:** **JWT** (roles comensal `diner` / restaurante `owner`).
- **Deploy:** Vercel/Render + Atlas, PWA — diferido.
- **Puertos:** API **:3013**, Mongo **:27018**, frontend Ionic **:8113**.

## Identidad visual (mockup 13-tastytable.html · tokens 13)
Apetitosa y elegante, cálida. **Terracota `#C25E3F`** (primario/CTA), **borgoña `#6E2B35`** (secundario/chips activos),
**disponibilidad `#4F8A5B`** (verde "disponible hoy"), fondo **crema `#FBF6EE`**, texto cocoa `#3A2C24`/`#6B5A4E`.
Display **Lora** (serif — nombres de restaurantes), body **Outfit**. Cards 24px, botones pill, **ticket con perforación**
y código de barras. 4 pantallas: descubrir (cards foto+rating+"disponible hoy"), detalle (galería+menú+selector de
reserva fecha/hora/personas/zona), confirmación tipo ticket, mis reservas. Fotografía de comida protagonista
(en demo: gradientes cálidos f-1…f-4 como placeholder de foto).
